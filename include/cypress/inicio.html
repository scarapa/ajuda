1º STARTANDO o projeto

Iniciar o projeto e a criacao do json 
$ npm init -y
OBSERVACAO, o conceito de dependencias de execucao e de desenvolvimento.
Esses conceitos ficam armazenados no package.json (devDependencies/dependencies)

Agora iremos abrir o arquivo (package.json) para editarmos 
$ code package.json
Alteraremos a linha test 
"scripts": {
    "test": "cypress open"
  },

$ npm test
Após a instalacao , executar o Cypress 
$ cypress open

2º ARMAZENAMENTO DOS TESTES

São dos arquivos importantantissimos para que o teste funcione , commands.js e o segundo o nome que VOCE ESCOLHER

cypress\support\commands.js

Local onde ficam armazenados o modo e local de acesso ao(s) sistema(s), exemplo abaixo:

cypress\integration\01\NOME_DO_TESTE.JS

É recomendável que cria uma pasta para isolarmos nossos testes de outros testes, no exemplo, criamos a pasta 01

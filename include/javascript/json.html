A ideia utilizada pelo JSON para representar informações é tremendamente simples: para cada valor representado, atribui-se um nome (ou rótulo) que descreve o seu significado. Esta sintaxe é derivada da forma utilizada pelo JavaScript para representar informações. Por exemplo, para representar o ano de 2012, utiliza-se a seguinte sintaxe: 
"ano": 2012
Um par nome/valor deve ser representado pelo nome entre aspas duplas, seguido de dois pontos, seguido do valor. Os valores podem possuir apenas 3 tipos básicos: numérico (inteiro ou real), booleano e string. As Listagens 2, 3, 4 e 5 apresentam exemplos. Observe que os valores do tipo string devem ser representados entre aspas.
NUMERO 
"altura": 1.72
STRING 
"site": "www.devmedia.com.br"
BOOLEANO 
"casado": true
A partir dos tipos básicos, é possível construir tipos complexos: array e objeto. Os arrays são delimitados por colchetes, com seus elementos separados entre vírgulas. 
["RJ", "SP", "MG", "ES"]
Matriz de Inteiros 
[
  [1,5],
  [-1,9],
  [1000,0]
 ]
Os objetos são especificados entre chaves e podem ser compostos por múltiplos pares nome/valor, por arrays e também por outros objetos. Desta forma, um objeto JSON pode representar, virtualmente, qualquer tipo de informação! O exemplo abaixo mostra a representação dos dados de uma UM de carro. 
{
	"carros": [{
		"modelo": "meriva 1.4",
		"ano": null,
		"motor": "1.4",
		"valor": [{
			"min": 15.000,
			"max": 34.000
		}],		
		"imagem": "meriva_01.jpg"
	}]
}
É possível representar mais de um objeto ou registro de uma só vez. 
{
	"carros": [{
		"modelo": "Meriva 1.4",
		"ano": null,
		"motor": "1.4",
		"valor": [{
			"min": 15.000,
			"max": 34.000
		}],		
		"imagem": "meriva_01.jpg"
	},{
		"modelo": "Golf 1.8",
		"ano": null,
		"motor": "1.8",
		"valor": [{
			"min": 25.000,
			"max": 35.000
		}],		
		"imagem": "meriva_01.jpg"
	}]
}
Por fim, é importante citar que a palavra-chave “null” deve ser utilizada para a representação de valores nulos 
"site":null
